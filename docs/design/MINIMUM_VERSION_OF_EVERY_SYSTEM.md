# Minimum Version of Every System

## Definition of “minimum”

The first full skeleton is successful when every department participates in one deterministic economy through timber, money, location, time, or information. A system may be visually crude, but it may not fake conservation, cash timing, ownership, compatibility, or causal state changes. “Later” means richer behavior, not repairing a disconnected prototype.

All numeric assumptions live in versioned configuration with source metadata. `PH` means a centralized `PLACEHOLDER`; `RR` means `RESEARCH_REQUIRED`.

## Shared economic acceptance gate

Before the skeleton milestone is complete, one scenario must show: opportunity → committed cash → owned lot → batch split → dispatch → buyer gate/measurement → receivable → delayed payment → ledger/P&L, while a competitor consumes a finite opportunity and a regime/season change alters a later decision through causal state.
